Githubの認証方式が変更に！！！

 September 4, 2021に投稿  |  Kenji Arai

はじめに

久しぶりに、このgithubのブログページに記事をアップしようとしたら、エラーとなって、記事がアップできなかった。

githubの認証方式が変更になったようで、自分は、以下のように対応して、記事がアップできるようになった。

現象

Docker, singularityコンテナについてのブログ記事をアップしようと、記事を書き、シェルプロ（deploy.sh；詳細はこの記事を参考に）で記事をアップしようとしたら、次のようなエラー。

remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.
remote: Please see https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ for more information.
fatal: Authentication failed for 'https://github.com/akenji3/akenji3.github.io/'

どうやら、この8/13を以って、password認証の機能が削除されたようだ。上記のエラーメッセージにある通り、移行スケジュールがこのページで半年以上も前から案内されていた。

さてどうしたものかと、ネットで検索すると、この記事が参考になった。記事の最後に本家のページへのリンクがあり、その通りに個人アクセストークン（PAT）を得て、それを使った。

対応

上記で述べた本家のページでは、PATを得て、それを使って、git cloneで対応するまでを、画面イメージを使って丁寧に説明している。この通りに実行すれば失敗もないと思われる。

ただ、自分のように、git本来のソースをやりとり／管理のために使っていない場合は、最後の「git clone ・・・」は使えない。

直近にアップしていた記事をローカル側で削除し、再度deploy.shを実行すると、password入力が求められたので、その際に、上記の手順で得たPAT（40文字程度の文字列）を入力（コピペ）した。

その後、ローカル側で削除した記事を戻して、再びdeploy.shを実行すると、正常にアップされた。

最後に

暫くサービスを使っていないと、今回のように認証などが変更になり、？？？となることもあるけど、落ち着いて調べれば、先人の努力の跡があり、それを参考にして、なんとか対応できることに、感謝。
